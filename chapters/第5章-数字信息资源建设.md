## 【核心概念】

### ★ 必考概念

| 概念 | 定义 | 关键要点 |
|------|------|----------|
| **电子资源（IFLA定义）** | 需要通过计算机访问的资料，可通过互联网远程访问或在本地使用；常见类型有电子期刊、电子图书、全文数据库、索引文摘数据库、参考数据库、数值和统计数据库、电子图像、电子音频/视频等 | IFLA 2012年发布 |
| **四级数字资源结构** | 永久保存级 / 服务级 / 镜像级 / 链接级四个层次，体现数字馆藏的层次性与重要性分级 | 上海交大案例：4个级别（非常重要/比较重要/重要/一般） |
| **永久保存级馆藏** | 保存价值已得到鉴定、用途得到认可、具有唯一性，且仅定位在某一个特定图书馆，其他地方不可获得 | 最高级别，唯一性是核心 |
| **服务级馆藏** | 对用户有用且必须用的虚拟馆藏；本馆用户十分需要，但贮存在其他数字图书馆的服务器上 | 虚拟馆藏，存于他处 |
| **镜像级馆藏** | 其他数字图书馆馆藏的拷贝，地址可能慢慢移动并经常不可存取，在其他地方也很容易找到 | 拷贝性质，不稳定 |
| **链接级馆藏** | 在任何地方均可寄居的信息资料 | 最低级别 |
| **多重价格模式** | 通过不同角度或标准将数据库分割成不同产品，赋以不同价格，实现市场细分 | 4种细分方式 |
| **捆绑价格模式** | 将两种或两种以上的相关产品捆绑打包定价出售 | 3种捆绑方式 |
| **歧视性价格模式** | 数据库商为实现收益最大化，针对不同图书馆的需求和支付能力制定不同收费价格，形成价格歧视 | 4种分级方式 |
| **国家采购** | 对与国家经济及社会发展战略目标相关的、教学科研人员和社会公众普遍需要的基础性、综合性数字资源，利用公共财政即政府出资购买的采购模式 | 3型：统购/补贴/平台 |
| **集团采购** | 若干图书馆自愿组成集团，共同推举谈判代表与电子资源提供商谈判价格与使用条款，最终购买合同由提供商与各加盟馆签订，购买费用由各成员馆自行支付 | DRAA是典型代表 |
| **DRAA** | 高校图书馆数字资源采购联盟（Digital Resource Acquisition Alliance of Chinese Academic Libraries），由成员馆、理事会、秘书处组成 | 中国高校集团采购核心组织 |
| **单馆自主采购** | 各馆根据本馆需要，自发与信息资源提供商谈判价格并自行签署购买使用协议 | 适用于价格不高而利用率较高的数据库 |
| **数据资源化** | 数据收集与清洗、数据标注与描述、数据组织与集成，使数据成为可用资源 | 数据价值化第一阶段 |
| **数据资产化** | 数据确权、数据估值、数据管理，使数据成为可计量的资产 | 数据价值化第二阶段 |
| **数据资本化** | 数据保值增值、数据流通交易、数据开发利用，使数据成为可交易的资本 | 数据价值化第三阶段 |
| **高质量数据集** | 经过采集、加工等数据处理，可直接用于开发和训练人工智能模型，能有效提升模型表现的数据的集合 | 2024年12月首次提出 |
| **FAIR原则** | 科学数据开放共享应实现可发现（Findable）、可访问（Accessible）、可互操作（Interoperable）、可重用（Reusable），4个目标层层递进 | 2014年1月提出 |
| **数据论文（Data Paper）** | 以数据集为核心对象的新型学术论文，聚焦数据本身，强调数据的描述、验证和可复用性 | 开放科学重要形式 |
| **脏数据（Dirty Data）** | 在数据处理中，由于各种原因导致的不准确、不完整、不一致或重复数据 | 低质量数据 |

### ☆ 重要概念

| 概念 | 定义 | 关键要点 |
|------|------|----------|
| **电子馆藏体系结构** | 电子资源的构成成份及其所占比例，包括文种结构、学科结构、时间结构、等级结构 | 比例协调平衡 |
| **电子资源运作方式** | ①出版社/学会自行建设数据库 ②数据库代理商和中间商建设提供门户服务 ③制作与销售文摘索引数据库 | 3种运作方式 |
| **资源品质评价三角度** | ①资源内容（全、广、新）②检索系统功能 ③资源提供商的信誉与服务 | 采购品质评估框架 |
| **Stephen Rhind-Tutt定价模型** | 美国电子出版商将定价模型分为4大类，共提炼出50多种价格模型，组合派生几千种价格结构 | 多样性与复杂化 |
| **公共基础层信息** | 具有综合性、基础性特点的检索性、工具性数据库（文摘/索引/书目二次文献，词典/百科三次文献） | 如Web of Knowledge、Ei Village |
| **机构知识库（Institutional Repository）** | 集成本校优质学术资源，以学位论文、期刊论文、研究报告、技术报告、会议论文五种为主要收藏对象 | 开放存取+网络出版+永久保存+公益服务 |
| **学科信息门户（SBIG）** | Subject Based Information Gateways，完成某学科网络资源内容高度组织集成和网络应用程序聚集的网络服务 | 可定制个性化界面 |
| **网络信息采选四要求** | "特"（特色资源）/"精"（精品资源）/"全"（完整性）/"省"（利用免费资源） | 采选原则 |
| **元数据法** | 选用通用数据单元描述网络信息检索特征，使管理者和使用者了解并辨别资源 | DC元数据为代表 |
| **统一资源发现系统** | 基于庞大元数据集合，通过类似Google的简单检索框，采用统一路径实现对图书馆各种资源的发现和获取 | Summon/EDS/Primo等 |
| **政务数据** | 主要存在于政府电子政务系统，是政府公务活动过程中生产的数据所形成的数据资源 | 公共数据 |
| **科学数据** | 自然科学、工程技术科学等领域通过基础研究、应用研究、试验开发等产生的数据，以及通过观测监测、考察调查、检验检测等方式取得的原始数据及其衍生数据 | 《科学数据管理办法》2018年 |
| **数据清洗** | 通过清洗提升数据质量、可用性和合规性水平，按需求转化为高质量数据集 | 静态质量+动态质量 |
| **数据标注** | 对原始数据（图像、文本、音频、视频等）及其中的对象进行标记并添加结构性标签的过程 | 结构化标签 |
| **元数据（Metadata）** | 数据的数据，针对数据属性特征、管理特征、使用特征、保存特征等依据标准规范进行描述和揭示形成的原始数据信息化记录 | 数据描述 |
| **COUNTER标准** | 电子资源使用统计的国际行业标准，要求统计数据符合该标准并实现自动收割 | 行业标准参考，PDF未展开 |

## 【重点公式/算法】

> 本章以分类框架和流程为主，以下为需要记忆的关键数字、结构模型和分类体系。

### 1. 电子资源采购四依据

```
        馆藏政策 (A)
           │
用户需求 (B) ── 电子资源采购 ── 资源品质 (C)
           │
        定价模式 (D)
```

**记忆口诀**：政策定方向，需求定目标，品质定标准，价格定决策。

### 2. 四级数字资源结构

```
永久保存级 ──→ 服务级 ──→ 镜像级 ──→ 链接级
（最高）                                        （最低）
唯一性          虚拟馆藏       拷贝性质       寄居信息
不可获得        存于他处       不稳定         任何地方
```

**记忆口诀**：永服镜链，从高到低；永久唯一，链接最弱。

### 3. 定价模式3大类11种结构

```
电子资源定价模式
│
├─ 多重价格（4种）
│   ├─ ①全库价格 + 学科库价格（如爱思唯尔SD：全库+24学科库=25种价格）
│   ├─ ②现期数据价格 + 回溯数据价格（如《自然》系列：现刊限最近5年）
│   ├─ ③全库价格 + 单品种价格（如德古意特电子书；OCLC ECO按篇购买）
│   └─ ④数据库价格 + 印本价格（IOP模式 / Wiley模式）
│
├─ 捆绑价格（3种）
│   ├─ ①印本捆绑（最典型常见，如爱思唯尔SD最低订购额）
│   ├─ ②新产品捆绑（打包新产品，涨幅偏高）
│   └─ ③内容重复产品捆绑（如EBSCO ASP与BSP重复率20%）
│
└─ 歧视性价格（4种）
    ├─ ①按学校/机构规模分级（如汤森路透SCI：6个级别）
    ├─ ②按现有订购量分级（如Wiley：内容费越高，访问费越低）
    ├─ ③按使用统计分级（如IEEE按下载量4级；IOP按成本3级）
    └─ ④按使用需求分级（如SciFinder按并发用户数定价）
```

**记忆口诀**：多重四（全库学科、现期回溯、全库单品种、数据库印本），捆绑三（印本、新产品、重复），歧视四（规模、订购量、统计、需求），共计十一。

### 4. 三种采购方式对应关系

```
采购方式          层次结构              适用类型
国家采购    →    公共基础层信息        →    综合性和基础性
集团采购    →    综合性多学科全文层    →    综合性、多学科全文数据库
单馆采购    →    专业性边缘性学科层    →    专业性、边缘性学科数据库
```

### 5. 电子资源采购流程（10步）

```
①制定电子馆藏规划
        ↓
②出版信息搜集与经费预算
        ↓
③电子资源预评估 ──→ 不适合 ──→ 不购买
        ↓ 适合
④申请试用（试用期3～6个月）★
        ↓
⑤电子资源后评估（反馈式+补充式）
        ↓
⑥做出采购决策
        ↓
⑦确定购买方式并谈判价格 ──→ 谈判失败 ──→ 不购买
        ↓ 谈判成功
⑧签订许可协议或合同（18个方面标准化）
        ↓
⑨结算并提供正式服务
        ↓
⑩收集反馈意见和使用信息
```

**记忆口诀**：规划预算预评估，试用后评做决策，谈价签约结服务，反馈收集十步走。

### 6. 数据价值化过程（三阶段）

```
数据资源化          数据资产化          数据资本化
（使数据可用）      （使数据可计量）    （使数据可交易）
    │                  │                  │
    ├─ 数据收集与清洗   ├─ 数据确权         ├─ 数据保值增值
    ├─ 数据标注与描述   ├─ 数据估值         ├─ 数据流通交易
    └─ 数据组织与集成   └─ 数据管理         └─ 数据开发利用
```

### 7. 数据资源类型分类

```
数据资源
├─ 公共数据
│   ├─ 政务数据（政府公务活动产生）
│   │   ├─ 领域数据资源（如医疗健康/农业/科学数据）
│   │   └─ 区域数据资源（如上海/贵州大数据资源）
│   └─ 科学数据（基础研究/应用研究/试验开发产生）
│       └─ 原始数据 + 衍生数据
└─ 私密数据
    ├─ 企业数据（国有企业：电信/银行/央企）
    └─ 个人数据（直接标识/间接标识）
```

### 8. 数据清洗方法体系

```
数据清洗方法
├─ 缺失值处理（删除法 / 填充法）

├─ 格式标准化（正则表达式应用 / 数据类型转换）
├─ 去重处理（精确匹配去重 / 模糊匹配去重）
└─ 合规性处理（个人信息脱敏 / 保密信息密级设置 / 非法信息处理）
```

## 【方法比较】

### 1. 三种采购方式对比 ★★★

| 比较维度 | 国家采购 | 集团采购 | 单馆自主采购 |
|----------|----------|----------|--------------|
| **含义** | 政府出资购买基础性、综合性数字资源 | 若干图书馆自愿组成集团共同谈判 | 各馆自发与提供商谈判签约 |
| **资金来源** | 公共财政（政府出资） | 各成员馆自行支付 | 各馆自行支付 |
| **优势** | 缓解地区间、行业间数字鸿沟；提供平等访问权；统一市场获价格优惠；有效利用政府财政；利于资源整合和长期保存；促使电子期刊与印刷期刊分离 | 享受批量购买优惠；节省自行谈判的时间和人力成本；加强同地区、同行业馆际交流与合作；在资源评估、组织试用、改善服务等方面争取更多利益 | 针对性强采集本馆所需资源；便于形成本馆特色资源体系；可定制个性化服务和培训；对电子资源剔除自主性强 |
| **不足** | 采选电子资源针对性不强；削弱基层图书馆自主选择权 | 成员馆分散，联系沟通困难；各馆经费调整、人员变更、技术故障等不确定因素产生系统风险 | 资源采集容易陷入盲目境地；不利于应对数据库商肆意提价；电子资源长期保存代价较高 |
| **适用类型** | 综合性和基础性的公共基础信息 | 综合性、多学科全文数据库 | 专业性、边缘性学科数据库 |
| **对应层次** | 公共基础层信息 | 综合性、多学科全文数据库层 | 专业性、边缘性学科数据库层 |
| **典型实例** | 我国以国家自然科学基金委和国家科技图书文献中心为主体 | DRAA、CALIS、CSDL、Springer Link集团 | CNKI系列数据库、万方数据资源系统 |

**演变逻辑**：国家采购（宏观基础保障）→ 集团采购（中观协同议价）→ 单馆采购（微观特色补充），三者互补而非替代。

### 2. 国家采购三类型对比

| 类型 | 具体方式 | 经费来源 | 典型国家 |
|------|----------|----------|----------|
| **统购型** | 国家利用中央经费统一购买电子资源，全国每一台联网计算机均可免费访问 | 中央经费全额 | 冰岛 |
| **补贴型** | 政府支付一部分电子资源费用，另一部分由各个图书馆支付，资源使用对象覆盖全国大多数科研与高等教育单位 | 政府+各馆分担 | 加拿大 |
| **平台型** | 国家投资建立统一的电子资源采购与访问平台，平台运转经费由国家支付，但资源购买费由基层图书馆负担 | 国家出平台费+基层出资源费 | 英国 |

### 4. 四级数字资源结构对比

| 级别 | 名称 | 核心特征 | 获取方式 | 稳定性 |
|------|------|----------|----------|--------|
| **第一级** | 永久保存级馆藏 | 保存价值已鉴定、用途得到认可、具有唯一性，仅定位在某特定图书馆 | 本馆永久保存 | 最高 |
| **第二级** | 服务级馆藏 | 对用户有用且必须用的虚拟馆藏，贮存在其他数字图书馆服务器上 | 远程访问 | 较高 |

<details>
<summary>点击查看答案与解析</summary>

**答案：**

**四级数字资源结构**是电子馆藏体系等级结构的一种分类方式，按照数字资源的重要程度和保存方式分为四个级别：

1. **永久保存级馆藏**：保存价值已得到鉴定；用途得到认可；具有唯一性；并仅仅定位在某一个特定的图书馆，其他地方不可获得。这是最高级别的数字馆藏。

### 题目2（简答题）

**简述电子资源定价模式的3大类11种，并举例说明。**

<details>
<summary>点击查看答案与解析</summary>

**答案：**

电子资源定价模式呈现多样性和复杂化特点，可归纳为3大类共11种：

**一、多重价格模式（4种）**

通过不同角度或标准将数据库分割成不同产品，赋以不同价格，实现市场细分：

1. **全库价格+学科库价格**：将数据库全部内容定价，再按学科拆分定价。如爱思唯尔SD数据库，定价分为全库和24个学科数据库，共计25种价格。
2. **现期数据价格+回溯数据价格**：按现期数据（多为最近10年左右）和回溯数据拆分定价。如《自然》系列期刊数据库，现刊数据滚动限定为最近5年。
3. **全库价格+单品种价格**：允许购买全部内容或按名称购买单一品种，可与用户驱动采购结合。如德古意特电子书数据库；OCLC ECO数据库按篇购买。
4. **数据库价格+印本价格**：将数据库和印本分开定价，包括"电子版为主+印本价格"模式（如IOP含10%印本费用）和"内容价格+电子访问费+纸本印刷费"模式（如Wiley）。

**二、捆绑价格模式（3种）**

将两种或两种以上相关产品捆绑打包定价出售：

1. **印本捆绑**：最典型常见，订购电子期刊数据库时必须保持印本期刊订购。如爱思唯尔SD的"最低订购额+内容费+自由全文库访问费"。
2. **新产品捆绑**：出版社增加新产品时不单独销售，与原产品打包捆绑，价格涨幅偏高。
3. **内容重复产品捆绑**：重复度很高的产品捆绑销售。如EBSCO的ASP和BSP全文期刊重复率达20%左右，不单独报价而是捆绑定价。

**三、歧视性价格模式（4种）**

数据库商针对不同图书馆的需求和支付能力制定不同收费价格：

1. **按学校/机构规模分级**：根据最终用户数量及学术水平分级定价。如汤森路透SCI按学生总数、教师总数、博士点数、硕士点数等划分6个级别。
2. **按现有订购量分级**：根据图书馆现有期刊订购量分级。如Wiley电子期刊，内容费越高，电子访问费越低。
3. **按使用统计分级**：依据总使用量（如IEEE/IET按前三年下载量划分4级）或总使用成本（如IOP按单篇下载成本划分3级）分级。
4. **按使用需求分级**：依据需求大小定价，如SciFinder按并发用户数量定价，同级别院校并发用户越多费用越高。

**解析**：本题是简答题高频考点。答题时需准确写出3大类的名称和各类下属的具体种类数量，并配以典型实例。注意4+3+4=11的总数关系，以及多元化价格模式是上述多种模式的综合（如爱思唯尔SD同时包含多重、捆绑、歧视性价格）。

</details>

### 题目3（简答题）

**比较国家采购、集团采购和单馆自主采购三种电子资源采购方式。**

<details>
<summary>点击查看答案与解析</summary>

**答案：**

| 维度 | 国家采购 | 集团采购 | 单馆自主采购 |
|------|----------|----------|--------------|
| **含义** | 政府出资购买基础性、综合性数字资源 | 若干图书馆自愿组成集团共同谈判 | 各馆自发与提供商谈判签约 |
| **优势** | 缓解数字鸿沟；统一市场获价格优惠；有效利用财政；利于资源整合和长期保存；促使电子期刊与印刷期刊分离 | 享受批量优惠；节省谈判时间和人力成本；加强馆际交流合作；在资源评估、试用、服务方面争取更多利益 | 针对性强；便于形成特色资源体系；可定制个性化服务；剔除自主性强 |
| **不足** | 针对性不强；削弱基层图书馆自主选择权 | 成员馆分散沟通困难；经费调整、人员变更、技术故障等不确定因素产生系统风险 | 容易陷入盲目采集；难以应对数据库商肆意提价；长期保存代价较高 |
| **适用类型** | 综合性和基础性的公共基础信息 | 综合性、多学科全文数据库 | 专业性、边缘性学科数据库 |

**国家采购**又分为三种类型：统购型（国家全额出资，如冰岛）、补贴型（政府与各馆分担，如加拿大）、平台型（国家建平台，基层出资源费，如英国）。我国国家采购始于1997年，以国家自然科学基金委和国家科技图书文献中心为主体。

**集团采购**的典型代表是DRAA（高校图书馆数字资源采购联盟），组织形式包括地区集团、行业集团和复合模式。

**单馆自主采购**适用于价格不高而利用率较高的数据库，我国大部分中文数据库采用此模式。

### 题目5（名词解释）

**请解释FAIR原则。**

<details>
<summary>点击查看答案与解析</summary>

**答案：**

**FAIR原则**是科学数据开放共享的指导原则，于2014年1月在"联合共建数据公平港口"（Jointly Designing a Data FAIRport）学术研讨会上提出。该原则倡导科研活动产出的数据在开放共享过程中应努力实现四个目标，且层层递进：

- **F——可发现性（Findable）**：(元)数据被分配全局唯一且永久持久的标识符，用丰富元数据描述数据，在可搜索资源中注册或索引。
- **A——可访问性（Accessible）**：(元)数据可通过标识符使用标准化通信协议检索，协议应开放、免费和普遍可执行，即使数据不再可用元数据也应可访问。
- **I——可互操作性（Interoperable）**：(元)数据使用正式、可访问、共享和广泛适用的语言表示知识，使用遵循FAIR原则的词汇，包括对其他(元)数据的限定引用。
- **R——可重用性（Reusable）**：(元)数据描述丰富，发布时具有清晰可访问的数据使用许可证，与详细来源相关联，符合领域相关的社区标准。

FAIR原则表示科学共同体中每个利益相关者获得数据资源的机会应该平等一致，科学数据资源应该在科学共同体中公平流通。

**提出背景**：科学数据资源总量持续积累；科学研究进入数据驱动时代；科学数据共享成为全球共识。

**解析**：本题是名词解释高频题。答题时需写出FAIR四个字母的英文全称和中文含义，强调"四个目标层层递进"的特点，并简述提出背景。注意F/A/I/R的顺序和准确翻译。

</details>

### 题目6（简答题）

**简述数据清洗中静态质量与动态质量的区别。**

<details>
<summary>点击查看答案与解析</summary>

**答案：**

数据质量评价可采用静态和动态两种方法来度量，对应不同的数据清洗要求：

**一、静态质量**

- **关注焦点**：主要关注数据本身的关键属性。
- **基础指标**：准确性、完整性、一致性、时效性。
- **增加维度**：多样性、真实性、合规性。
- **评估重点**：数据的领域覆盖、来源可靠性，以及在隐私保护和安全合规方面的表现。
- **对应清洗**：面向静态质量的数据清洗包括脱敏处理、处理缺失值/重复值/异常值、规范数据形式。

**二、动态质量**

- **关注焦点**：强调数据集在模型训练和应用中的实际效果。
- **评价方法**：可通过引入代表性模型开展基准测试，结合基准评测数据集与量化指标，客观衡量模型性能的提升程度。
- **对应清洗**：面向动态质量的数据清洗根据数据使用需要和反馈，确定有针对性的清洗要求。

**核心区别**：静态质量是"数据本身好不好"的属性层面度量，关注入库前的质量把关；动态质量是"数据用起来好不好"的应用层面评估，关注投入模型训练后的效果。

**解析**：本题是对比题高频考点。答题时需分别说明两者的关注焦点、评价指标、评估重点及对应的清洗方法，并指出核心区别。注意静态质量有"基础指标+增加维度"的层次结构，动态质量强调"模型训练中的实际效果"。

</details>

### 题目7（论述题）

**论述电子资源的采购流程，并说明各环节的关键要点。**

<details>
<summary>点击查看答案与解析</summary>

**答案：**

电子资源采购涉及多种因素且价格昂贵，需要不断协商、反复论证的决策过程，在采购前、采购中和采购后都遵循一定规范，共包括10个步骤：

**一、采购前阶段（步骤1-3）**

1. **制定电子馆藏规划**：通过馆藏发展政策厘清本馆及所在机构特点、现状和用户需求，建立馆藏信息资源发展目标和基本原则。

2. **电子资源出版信息的搜集与经费预算**：信息搜集包括出版情况、资源内容信息、提供商信息、知识产权信息等；预算是一财政年度内收入与支出数额的估计，帮助合理控制经费。

3. **电子资源的预评估**：对资源内容、检索系统功能、提供商信誉与服务进行初步评估。如不适合则不购买。

**二、采购中阶段（步骤4-8）**

4. **申请试用**：电子信息提供商大都提供订购前的免费试用，**试用期3～6个月不等**。试用期间应指派专人负责测试和维护，监测使用情况，统计用户数量、使用频次、全文下载量，收集反馈意见，全面测试数据库质量。

5. **电子资源的后评估**：试用之后的反馈式和补充式评估，收集用户试用体会，聘请专家学者参与资源选择，对电子资源进行查重处理，据此调整订购决策。

6. **做出采购决策**：注意印刷型和电子版文献的关系；注意不同电子版的优选；注意馆际协调，避免"大而全""小而全"。

7. **确定购买方式并谈判价格**：根据资源层次和本馆情况选择国家采购、集团采购或单馆采购。如谈判失败则不购买。

8. **签订许可协议或合同**：许可协议是对买卖双方权利与义务的规定。重点考虑因素包括：①利用电子资源的权利与限制 ②永久利用问题 ③购买方式 ④印刷文本停订 ⑤合同文本语言 ⑥元数据、馆际互借、电子教参、机构库、存档、统计报告等。合同内容标准化应至少涵盖价格、资源质量、使用方式等**18个方面**。协议文本须"中英文同等有效"或中文，适用法律须为"适用中国法律"。

**三、采购后阶段（步骤9-10）**

9. **结算并提供正式服务**：跟踪出版商对订单的执行情况，提供配套服务包括代查代检、文献传递、数据库再开发、使用培训等。

10. **收集反馈意见和使用信息**：不断收集用户反馈意见、分析数据库使用统计数据，评价采购工作成效，为续订、停订、改订、新订提供借鉴，也为电子馆藏规划调整提供参考。

**解析**：本题是综合性论述题。答题时需清晰呈现10个步骤的逻辑顺序，分为采购前、采购中、采购后三个阶段，并突出关键要点（如试用期3-6个月、合同标准化18个方面、中英文同等有效等）。注意流程中的决策分支点（预评估不适合则不购买、谈判失败则不购买）。

</details>

### 题目8（简答题）

**简述网络信息资源采选的要求和网络信息组织的主要方法。**

<details>
<summary>点击查看答案与解析</summary>

**答案：**

**一、网络信息资源采选要求**

网络信息资源采选要对大量的网络原始信息以及经过加工的信息材料进行筛选和判别，有效剔除不需要的信息，选取所需要的信息。采选有四个要求：

- **"特"**：要搜集具有特色的网络资源。
- **"精"**：要搜集来源可靠、内容先进、背景明确、发布规范的精品资源。
- **"全"**：要保证搜集目标的完整性。
- **"省"**：要尽可能地利用免费资源。

学科信息门户（Subject Based Information Gateways，SBIG）是重要的采选工具，它提供网上大量网站或文献的链接服务，具有智能化的资源选择、智能化的内容描述和智能化的产生浏览结构/分类功能。

**二、网络信息组织方法**

主要有四种方法：

1. **分类法**：按照学科性质，根据学科概念体系，依学科分类方法组织网上信息，用户可按树状结构逐步找到所需信息。具有严密的系统性和良好的可扩充性，但不适合建立大型综合性资源系统，仅在建立专业性或示范性信息资源体系时显出优势。

2. **主题法**：以自然语言即文献作者或文摘作者原来使用的语言（包括关键词、自由词和出现在题名、摘要或正文中的语词），经过规范处理后直接作为文献主题标识，并按字顺排列组织与揭示信息。揭示信息直接、客观，利用词汇关系链获取领域知识。

3. **元数据法**：选用一定数量的通用数据单元来描述网络信息的检索特征，使管理者和使用者可以通过元数据了解并辨别资源。DC元数据基于资源发现为目的，为网络信息资源的著录提供一种更简单的格式。

4. **知识图谱可视化**：通过可视化形式，以地图、图形、图片形式显示信息内容与关系，提供更加直观、便捷的信息呈现。

**解析**：本题考查网络资源建设的两个核心知识点。采选要求注意"特精全省"四个字的准确含义。组织方法需掌握四种方法的名称、原理和特点，尤其注意分类法的适用局限和元数据法中DC元数据的定位。

</details>

### 题目10（论述题）

**论述数据资源建设面临的主要问题，并结合实际分析其挑战。**

<details>
<summary>点击查看答案与解析</summary>

**答案：**

数据资源建设面临三大主要问题：

**一、数据的权属问题**

- **核心争议**：数据属于谁。关于数据的权属，目前在法律上还是空白，所能够参照的只有知识产权法和物权法。由于数据资源的独特性质，这些法律显然不适用于数据权属。
- **合理性探讨**：因为数据不是天然存在的，所以"数据应该属于数据的生产者"的说法比较合情合理。
- **面临的具体问题**：①当数据是有多个主体生产时如何界定数据的权属；②当生产的数据涉及国家秘密或公民隐私时如何界定数据的权属。

**二、数据资源的价值衡量**

- **难以计入资产**：数据资源很难将其计入固定资产或流动资产。
- **可低成本复制**：数据资源拥有者可以以极低的成本大量复制并传播数据资源。
- **不因使用而变化**：数据资源不会因为使用而减少，也不会因为不用而增值，使得数据资源价值不好衡量。

**三、数据资源的开放、开发问题**

- **价值与矛盾**：数据资源具有很强的原始性，具有很大的应用价值。
- **制约因素**：资源开放、开发受到信息保密、信息安全、信息技术等多方面的影响，投入大、见效慢、老化快。
- **环节复杂**：数据生产和汇聚的方向主要是"业务—部门—法人机构—区域／行业领域—全国"，环节较多，开放、开发主体和规范亟待确定。

**挑战分析**：这三大问题相互关联——权属不清导致价值难以衡量，价值难以衡量又制约了开放和开发。随着2023年国家数据局挂牌成立、2024年"高质量数据集"概念首次提出，数据资源建设正从探索走向规范化，但法律框架、价值评估体系和开放共享机制仍需完善。

**解析**：本题是综合性论述题。答题时需分别阐述三大问题的核心内容，并结合政策背景（国家数据局成立、高质量数据集）分析发展趋势。注意三大问题之间的逻辑关联，体现系统性思考。

</details>
